More Information on Git

Jul 20th, 2013

make changes to a file

 git add --plus what files you actually changed     # Stages our changes

 git commit _m "changed something"    (# Commits our changes)

 git fetch upstream

If changes on the branch we’re based on (master)

 git rebase upstream/master

If conflicts:

 git mergetool

 enter diff

 git rebase --continue

I also learned today that Fork is a Github thing not a git command.

Although this was not as easy as it looks we did some paired programming and learned how to modify a file, how to pull in upstream changes and then how to resolve the conflict.
